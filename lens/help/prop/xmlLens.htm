xmlLens
Allows you to create virtual columns stored as XML records
Instead of altering the database table everytime you have a new field to add, store multiple fields as XML in one field.

The following example will generate the following XML in the field XMLSTORE:

$lens->xmlLens = 'XMLSTORE^NAME32^TITLE';

And this is the XML generated in the XMLSTORE field when you create a new record:

<?xml version="1.0"?>
<ROW>
<NAME32></NAME32>
<TITLE></TITLE>
</ROW>

Two new fields NAME32 and TITLE will be seen in the phpLens grid. Any change in the two fields will overwrite the contents of XMLSTORE. The two new fields can be manipulated using any of the phpLens properties. The field type is set to CHAR automatically.

If a virtual field name ends in a 2 digit number, that is treated as the field length. So NAME32 will be treated as a 32 character long field. TITLE will be treated as a field with unknown length (-1).

There are some limitations:
1. You cannot filter nor search within a virtual field.
2. PhpLens cannot store the XML in an Oracle CLOB or BLOB.
3. Do not modify the XML storage column yourself.

Syntax
$lens->xmlLens = 'Column_to_Store_XML^XMLCol1^XMLCol2^XMLCol3';
